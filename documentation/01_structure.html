Intro

This section of the documentation will explain the basic scheleton of the page, the most esential part of the template, how to create content classes how to add backgrounds to them and how to generally build the structure of the page where all your content will later be added. 

This structure is essential to obtain a strong, sturdy and to avoid any error.

If you're a HTML pro, you won't even need to read the other documentations if you read and understand this part. We've done our absolute best and succeeded to make all elements copy and paste, but all these copy paste elements have to go inside a structure, that is vital to the functionallity of the item, so please, take the time and read the follow section.

If you're a new user or a medium user, we can guarantee and there is absolutely no question this documentation will cover every aspect of our product so you won't need anything else to fully customize our product to bring your concept to life. 

With this being said, we'll still give you all the help you need in case you struggle with a feature or in case we omitted something, but reading this documentation will make editing this item a piece of cake! 

 

Page Structure

<div id="page-transitions">  
    <div class="page-preloader page-preloader-dark">
        <img class="page-preload" src="images/preload.gif">
    </div>

    <div id="page-content" class="header-clear">
        <div id="page-content-scroll">
            <!-- This is where every content we'll be writing about goes. Remember it! -->
        </div>  
    </div>
</div>

 

The above element will generate the most important part of the product. The structure where everything will be placed in, and also, the elements that generate the scrolling of the page and the transitions of the AJAX animations.  

Line 1 creates the main hosing div. This will switch the page using an AJAX plugin called smoohState. This allows us to load pages much faster and give a more native effect when switching between pages. 

Line 2, 3, 4 are the page preloader. In line 2 you can replace the secondary class page-preloader-dark with page-preloader-light which will change the overlay of the preload to white. To replace the icon itself please replace preloade.gif in the images folder with a preloader you desire. If you wish to change the preloader dimensions, use CTLR+F on Windows or CMD+F on Macbook in your code editor in framework.css and locate .page-preloader img. You will find all required settings for the image there

Line 6 is the content class. This houses everything on the page including the power to move it around when needed using CSS3 transform effects. 

Line 7 houses the class within which all the content of your pages will be added. This class is crucial to enable scrolling of pages. This technique is called scrolling divs and it allows a much larger flexibility for sidebar templates and templates that will have animated elements. 

Line 8 is where you place your content.

Line 9 , 10, 11 close the classes. 

For those who need modifications to the AJAX system, we strongly recommend reading the SmoothState Developer Page. Advanced coding knowledge is required. Please note that customization services for elements to add extra features is not included in the support package as per Envato Terms of Service and will be charged at freelance rates. 

Page Structure Special Case ( Background Boxed Center Color )

Content with Background Color 
 
Contents with background colors are contents where the background image is visible. These are excellent when you want your page to have a more interesting design. This page has a different structure than usual. Let's get to the example 
 
<div id="page-transitions">
    <div clas="page-preloader page-preloader-dark">
        <img class="page-preloader src="images/preloader.gif">
    </div>

    <div class="bg bg-full"></div>

    <div id="page-content" class="page-content header-clear-large bg bg-cover bg-transparent">
        <div id="page-content-scroll">
            <div class="content content-margins bg-white">
                <!--Enter only containers here. Do not close this content-->
            </div>
        </div>

    </div>
</div>
This page is unique, meaning, a page like this alters the entire structure of the content. It allows you to use a boxed design but requires you to place all elements within the same content class. This may not be the best solution, so that's why this page is uniquely built in design. 
 
Almost the same code applies as above except a few things. The bg bg-full class that will set the background using the bg and will make it fullscreen by using the bg-full
 
The page-content id also has a secondary classes added to it.
 
header-clear-large - clears the distance to that white element
bg bg-cover bg-transparent - settings for the background positioning and alignment. 
 
All your content must now go inside the content content-margins bg-white content box. This will have left, right, top and bottom margins so the background is seen but also inner paddings. Due to this reason this page structure should not be used for fullscreen sliders and fullscreen elements. These are explained and built differently. 

 

 

 

Content Classes

The Content Class.

<div class="content SECONDARY_CLASS"></div>

The content class, is the mother classes. This class will house absolute all elements ( that are of course not fullscreen ). This class should be the house class of all your non full screen elements.

What does that content class do? It's simple! It creates a wrapper with a left and right margin to diferentiate you elements between them. In this class, elements can be added individually or within containers which we will be explaining next. 

Secondary Classes:

content_fullscreen : will make the content fullscreen

full-bottom / half-bottom / no-bottom / full-top / half-top will add a margin to the direction mentioned in the code to diferentiate the element from the following element. Most elements have this added inside them and containers have that by default, but there might be cases where you need to add this to keep the visual hierarchy beautiful.

The Content Center Fullscreen Class

Some pages require a big text, or something in the center, let's say the 404 page. This can be used one time at the top of the page. This element is built using JS and it requires positioning code that tends to use the processing power of your device. Indeed on devices like the latest iPhone and the latest Samsung that's not a problem, but some of your users may not have these devices, and on some lower end devices we've tested on, we realized the page loaded much slower when more than one center item is used. 

Let's get to the code

    <div class="content-center">
        <div style="width:XXpx; margin:0 auto;">
             <!-- Content Here -->
        </div>
    </div>
    <div class="coverpage-clear"></div>

The content center class is very simple to use. 

Line 1 is the content center class. This class must be at the top, the first class right under the preloader or the header. 

Line 2 is the class that will determine the width of the content. If you want to show a text, or a button, then you don't need to use the entire width of the screen and making it smaller makes it look better. We strongly recommend setting it at a 300px screen if you aim to target mobiles and tablets or to set it to 85% if you wish to tag mobiles and tablets as well. 

You'll be able to use the text and button classes to center these elements explained in their apporpirate chapters. 

Line 3 is where you add your content. This is limited to text, buttons and icons. We strongly recommend not using other elements. 

Line 4, 5 close the content elements

Line 6 is the coverpage-clear class. This is extremly important. The content-center class is an absolute element, so it needs a underlying element to adjust it's height to the entire size of the screen. Not adding this class will result in the content of the content-center element to be overlayed on the content under it.  After you close this, you can continue with your page by adding another, simple content class. 

Secondary Classes:

These classes must be added to the coverpage-clear class. 

full-bottom / half-bottom / no-bottom / full-top / half-top will add a margin to the direction mentioned in the code to diferentiate the element from the following element. Most elements have this added inside them and containers have that by default, but there might be cases where you need to add this to keep the visual hierarchy beautiful.

The Container Class

<div class="container"></div>

The container class is a very, very, very important class when using elements that JS targets individually. An instant example is toggles. When you tap a toggle, the JS searches for it's content in it's housing container and shows it. So having 2 of them in the same place will lead in a bug where both toggles get deployed by only tapping one. This is easily solved by having each toggle inside a separate container. 

Containers can also be used to include a better visual hierachy between elements, and make them easier to diferentiate themselves between other elements. 

Secondary Classes:

full-bottom / half-bottom / no-bottom / full-top / half-top will add a margin to the direction mentioned in the code to diferentiate the element from the following element. Most elements have this added inside them, but there might be cases where you need to add this to keep the visual hierarchy beautiful 

 

Responsive Classes

Using responsive classes, or floating classes is extremely easy. All the classes bellow will have a few things in comon. The class names are identical for each example below, and in each case to the last class a secondary class is added called last-column. This makes them take full advantage of the screen and work with removing the right margin. At the end of each, there is a class that must be added to achieve better compatibility with certain browsers. The clear class. This should be added after any floating element. 

Two Columns ( on all screens )

<div class="one-half"><!-- Content Here --></div>
<div class="one-half last-column"><!-- Content Here --></div>
<div class="clear"></div>

Three Columns ( on all screens )

<div class="one-third"> <!-- Content Here --> </div>
<div class="one-third"> <!-- Content Here --> </div>
<div class="one-third last-column"> <!-- Content Here --> </div>
<div class="clear"></div>

Two Columns ( on screens larger than mobile  )

<div class="one-half-responsive"><!--Content Here--></div>
<div class="one-half-responsive last-column"><!--Content Here--></div>
<div class="clear"></div>

Three Columns ( on screens larger than mobile )

<div class="one-thrid-responsive"><!---Content Here--></div>
<div class="one-third-responsive"><!---Content Here--></div>
<div class="one-third-responsive last-column"><!--Content Here--></div>
<div class="clear"></div>

Four Columns ( on screens larger than mobile )

<div class="one-forth-responsive"><!--Content Here--></div>
<div class="one-forth-responsive"><!--Content Here--></div>
<div class="one-forth-responsive"><!--Content Here--></div>
<div class="one-forth-responsive last-column"><!--Content Here--></div>
<div class="clear"></div>

The columns also allow using secondary classes for margins mentioned in the previous chapters. 

 

Classes and Secondary Classes

Probably the most important add-on to most of the features in our template are pre-coded, simple built secondary classes. Secondary classes are basically any class that can be added to a HTML element to extend it's native capability. Let's give a quick example 

<p> I'm a small text </p>

What you see above is a basic, simple paragraph, nothing special about it. If you'd want to center it, or change it's color, you'd have to open the CSS, find the matching color, search for font sizes and do all of those, right? Well, yes and no. We have a tone of pre-coded classes built to help you out with just these things. 

<p class="center-text larger-text color-red-dark uppercase"> I'm a centered larger text with the color red in uppercase now<p>

Chances are you won't need these many secondary classes, but adding an extra line of CSS just to center a text, or change it's color would be pointless, so these elements are added in here to ease your work and also ease the load on the CSS which has been optimized to perform as fast as it can for it's current load. 

Let's get to showing you the secondary classes. 

(1) - (universal) - can be added to all elements 

(2) - (individual) - can be added to a single element, for example, a text, a heading, a button, an anchor.

Unqiue secondary classes to each element will be specified when you meet them in each documentation, adding them here without their apropriate code structure would be pointless as seeing the code will make more sense than just seeing the class. 

hide-if-mobile  - (1) -  will only show content on larger screens 

hide-if-responsive - (1) -  will only show content on mobile screens

uppercase / lowercase / thin / thiner / bold / ultrabold - (2) - will alter text formats. 

small-text / smaller-text / normal-text / larger-text / large-text - (2) will alter text sizes.

center-text / left-text / right-text - (2) will alter text alignment

center-text-mobile / left-text-mobile / right-text-mobile - (2) - alters text alignemnt only on mobiles

center-text-tablet / left-text-tablet / right-text-tablet - (2) alters text alignement only on tablets

rotate-45 / rotate-90 / rotate-180 - (2) - will alter the rotation of icons. Very useful for font-awesome

full-top / half-top / no-top - (1) - alters the margin / distance  from the above element to match visual hierarchy

full-bottom/ half-bottom / no-bottom - (1) alters the margin / distance from the above element to match visual hierarchy

no-border - (1) - removes the border of elements that have it.

responsive-image - (2) - adding this class to any image will make it stretch to the screens full size.

 

These are the classes that affect contents and indivudual text elements and can be used universally for these elements.
