FAQ

What are these starters I keep seeing?

Each starter represents an empty template to begin with, having the components in the name present.

How do I install the item?

Copy and paste all the items in the code folder to your server to your desired location. 

What are those i class"fa " things?

That is Font Awesome, the amazing icons we used for this item you can learn more about them and find a cheatsheet on how to use them here http://fortawesome.github.io/Font-Awesome/icons/

How do I change the contact form information?

There is only one PHP file in the item, please open that item and replace the email address with yours. 

How do I make the login page / sign in page / calendar page / news / store / blog  to work? 

Please remember that this is a site template and is not connected to any CMS or any databases, so these features are mearly template designs, they are HTML and CSS and need to be connected to a back-end to work. 

 What is the Mobile UI section?

The mobile UI section is comprised of elements that you can use to create an app. Please note that converting the item to an app is entirely up to you. If these elements don't suit you and you wish to reduce the page load, simply open framework.css and scroll to the very bottom. You can remove them from there. 

I want to add my own JS, it's not working, why?

Add any JS you want starting from line 4. THis is very important since the AJAX refreshes the functions and they are present starting with line 3 and down to 955. When you tap a link that entire zone is refreshed. Be sure to import your script in all pages you want it to work. My suggestion is to copy the min version of your script in plugins.js and import it there as we imported ours. 

This uses AJAX? How to I use a simple link? They don't work! 

Any link that points outside the main item should have the class default-link. This will terminate the smooth state / ajax page loading for that link and it will work like anyother regular link! 

I don't want AJAX. I want the old school loading style. How? 

That's actually very easy. Open custom.js and delete lines 956 to 998

I'm a tech guru. I want more API's for the AJAX. I need them. 

Everything you need to know to customize the AJAX and more API's can be located on the plugin page over at https://github.com/miguel-perez/smoothState.js

I want to build an app using PhoneGap and Cordova. Can you help me? 

We only provide support for the item as is, meaning a site template. PhoneGap and Cordova's pages state clearly that they work wiht ANY html, css and JS item on the planet. So compatiiblity is not an issue. Furthermore, we use the latest and easiest code techniques to assure there won't be issues. 

Do I need an extended license?

If your app will be paid or have in-app paid features, yes. You do. We will pursue legal action against any app on any application store that uses our code and can't provide a license for an extended license use. 

Header Clear

Clearing the header / showing content so it's not under the header is quite easy.

<div id="page-content" class="SECONDARY_CLASS"><!--Read the structure, it goes here--></div>

On the main ID class where all your content goes, meaning, page-content, add the secondary class called header-clear to clear the exact size of the header in case of sliders and header-clear-large to clear another 30 pixels so there's some space between them

    <a href="#" class="header-icon-1 show-left-sidebar"><i class="fa fa-navicon"></i></a>
    <a href="#" class="header-icon-2 show-header-menu"><i class="fa fa-angle-down"></i></a>

Let's explain everything in basic human words, so you don't get headaches trying to understand this. It's quite VERY simple! 
 
The header is comprised of 5 main elements. 4 icons and the logo. Each element is placed using the first class, header-icon-NUMBER while the second class dictates the activation for that element. For example, heading-icon-1 in our case will deploy the left sidebar, but, if you want to use it as a link, you just remove the secondary class and fill in the href. 
 
Replacing the logo is simple as well. You can locate the header-logo class inside style.css on line 184. Replace the image there with yours and make sure to change the widht, height and background-size parameteres to half of your image to maintain aspect ratio. 
 
You can choose between the following secondary classes header-dark / header-light / header-transparent

The material menu can house up to 6 icons. This is done because of the fact that mobile devices have a limited available height, and although you may be rocking the biggest Note or Iphone out there chances are some of your users still have smaller devices. Using Skins

Skins are incredibly easy to use and to switch in betwee, by default, we have 3 skin files included. skin_dark / skin_light / skin_material. To activate any of these skins, simply name it skin in the styles folder. 

The dark and light skin have no special classes of their own, they simply reskin the entire page where that css is used, but the material skin being a bit more different needs some special classes to work with. These are very easy, and only require you to play with secondary classes on the content class

Using the Material Skin

As you've noticed the item's interior is split into content classes. Each content class by default will be turned into a material white backgrounded box with a shadow, the typical design of material. But there are some econdary control classes to better arange and edit it according to your needs 

<div class="content SECONDARY_CLASS">
      <!--  Your Content Here -->
</div>

The secondary classes offer you control offer the following

material-no-bottom  / material-no-top - removes the content bottom/top in case your elements inside already have one provided

material-half-bottom / material-half-top - adds only half the bottom/top of the content

material-full-bottom / material-full-top - adds the entire bottom/top of the content the bottom of it

no-material - removes the material design of the content

content-fullscreen-material - extends the box to the max left andmax right width of the screen without hiding the bottom and top shadow. this should be added wihtout the initilal content class or with the content-fullscreen class
