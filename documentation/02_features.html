Content Show/Hide

This section covers toggles, accordions, tabs, dropdown menus or anything that affects content and requires interaction to deploy. 

Accordions

<div class="accordion-item SECONDARY_CLASSES">
    <a href="#" class="accordion-toggle">Accordion Item <i class="fa fa-plus"></i></a>
    <div class="accordion-content ">
        <!-- Your Content Here-->
    </div>
</div>

Be sure to keep your each accordion item inside it's own container / content tag so the JS can target it properly.

The accordion is comprised of 3 major parts

Line 1 is the accordion holder and allows a set of secondary classes for styels 

Line 2 is the accordion deploy button

Line 3 is the accordion content. 

No secondary class - will result in an accordion with borders around it

accordion-ghost  - will result in an accordion that expands full width, with no borders around it

accordion-bg bg-VALUE-VALUE  - will change the color of the accordion background. You can use any of the colors mentioned in the first chapter.

Tabs

<div class="content">
    <ul class="tabs">
        <li class="tab-link current SECONDARY_CLASS" data-tab="tabs-1">Tab 1</li>
        <li class="tab-link SECONDARY_CLASS" data-tab="tabs-2">Tab 2</li>
    </ul>

    <div id="tabs-1" class="tab-content current SECONDARY_CLASS">
        <div class="tab-elements">
              <!--Tab Content Here-->
        </div>
    </div>
    <div id="tabs-2" class="tab-content SECONDARY_CLASS">
        <div class="tab-elements">
             <!--Tab Content Here-->
        </div>
    </div>
</div>

Be sure to keep your each tab group item inside it's own container / content tag so the JS can target it properly.

The tabs are made using a very easy to use construction. Each tab content is an individual ID with the the name identical to the data-tab of the list that targets it. 

Tab titles can be replaced with icons and the secondary classes available allow you to use all the background colors mentioned in the first chapter to style them.

 

 

 

 

 

 

 

 

 

 

 

 

 

Device Detection

Detecting devices are extremly useful for your items especially if you're trying to target a designated audience. 

To show specific content for a device style only the usage is very simple. Simpy add the following classes to any content element. 

show-android - will show content only for android

show-ios  - will show content only for ios 

show-windows - will show content only for windows mobile

Dividers

Divideres are decoration elements that can be placed within your page to set a border between elements. You can change them easily and they set a bottom margin that's equal to the hierarchy of the pages. 

Dividers are, by default, expanding to the full width of the wrapper they are housed in. If they're placed within containers or content classes, they will have equal margins to the left and right of the page as the content. 

If they are placed outside the content classes, they will expand to the full widht of the screen. 

<div class="decoration deco-0 SECONDARY_CLASS"></div> <!-- Creates a Red Line -->
<div class="decoration deco-1 SECONDARY_CLASS"></div> <!-- Creates a Green Line --> 
<div class="decoration deco-2 SECODNARY_CLASS"></div> <!-- Creates a Blue Line -->
<div class="decoration deco-3 SECONDARY_CLASS"></div> <!-- Creates a Gray Line -->
<div class="decoration deco-4 SECONDARY_CLASS"></div> <!-- Creates a Yellow Line --> 
<div class="decoration deco-6 SECONDARY_CLASS"></div> <!-- Creates a Magenta Line --> 
<div class="decoration deco-7 SECONDARY_CLASS"></div> <!-- Creates a Black Line -->
<div class="decoration deco-8 SECONDARY_CLASS"></div> <!-- Creates a Teal Line -->
<div class="decoration deco-9 SECONDARY_CLASS"></div> <!-- Creates a Dark Gray Line -->

Using the default class decoraiton without any deco secondary classes will create the default small line. 

The secondary class that can be used is decoration-margins and this is easy to use if you plan on using outside of content classes where the decoration will expand to full width. If you use decoration-margins will add a left and right margin. 

Other Decoration Styles 

If you want a shadow effect added for the etnire width of the screen you can use 

<div class="DECO_CODE"></div>

Replace deco_code with decoration-shadow / decoration-line / decoration-zig-zag / decoration-slash to use other styles of decorations

 

Dropdowns

Dropdown menus can be used to implement a submenu or menu directly inside a page. The menus can be and must use color codes to define them. 

<div class="dropdown-menu">
    <a href="#" class="dropdown-item dropdown-toggle SECONDARY_CLASS"><i class="fa fa-home"></i><em>Dropdown</em><i class="fa fa-plus"></i></a>
    <div class="dropdown-content SECONDARY_CLASS">
        <a href="#" class="dropdown-item"><i class="fa fa-clock-o"></i><em>Sub Item</em><i class="fa fa-angle-right"></i></a>
        <a href="#" class="dropdown-item"><i class="fa fa-star"></i><em>Sub Item</em><i class="fa fa-angle-right"></i></a>
    </div>
</div>

The dropdown menu is comprised of a main anchor that deploys the internal list of menus. To create more dropdown menus simply duplicate the anchors inside the dropdown-content and replace the icon and text with your own.

Secondary Class

The secondary classes must be replaced with background colors described in the first chapter. We recommend using the dark version of the background color for the first secondary class and the light version of the background color for the second class for a better visual effecft.

Inputs

Checkboxes & Radios

The FAC means Font Awesome Checkbox, because we use font awesome for the icons to reduce load on your page and also make things load much much faster. Checkboxes come in different color styles. We'll explain how each style can be achieved. 

<div class="fac fac-checkbox fac-default"><span></span>
    <input id="box1-fac-checkbox" type="checkbox" value="1" checked="">
    <label for="box1-fac-checkbox">Simple</label>
</div>

The main div houses the default input and label used for checkboxes. The label and id must be unique from other input and labels but identical inside the same checkbox div to target eachother. 

The checkbox div has 3 classes. 

fac - general styles

fac-checkbox - which can be switched to change the style

fac-default - to change the color 

 

Replace fac-checkbox with: 

fac-checkbox-round / fac-radio / fac-radio-full / fac-checkbox-o / fac-checkbox-round-o  to change the style of the icon to the left of the text. 

 

Replace fac-default with: fac-blue / fac-green / fac-orange / fac-red to change the color of the icon to the left of the text.

Please remember. Each checkbox and radio must be placed in it's own div, and each input and label for that div must have the same ID and FOR in order to match them when tapping / clicking.

Input Fields

Just copy and paste classes each with a different style and different purpose

<div class="input-icon">
    <input type="text" class="input-text-box input-red-border" value="John">
    <i class="fa fa-user"></i>
</div>

The main class can be set to input-icon, if this is added, there will be an icon at the right side of the input field. The icon can be replaced using the classic font awesome icons

Each input form can have different parameters to set the border style and border color

Replacing input-text-box with input-text-line will replace the entire border with only a bottom border.

Replacing input-red-border with input-green-border will replace the entire box border with a red or green border on focus. 

Replacing input-red-line with input-green-line will replace the bottom line with a red or green line on focus.
